Deep dive · 01

Components form themselves.

The defining difference from Figma and everything like it: in Holon, you never declare a component. Structure is the only ceremony.

The problem

Componentization is a chore, so nobody does it

In canvas tools, a component is a decision you have to make — select, name, declare, maintain. You must guess up front what deserves to be reusable, and the ceremony is heavy enough that most designs end up as ungrouped soup: hundreds of loose layers pretending to be structure.

And once declared, the component library and the actual design drift apart. The diagram rots while the pixels move on.

The approach

One law replaces the ceremony

Holon derives components from the structure you actually build. There is no “componentize this” ritual, no naming dialog, no library housekeeping.

An element with no children — a lone text, an empty div — is just an element. The moment it contains something, it becomes a component: openable, editable, versionable, on its own.

If an element has a child, it is a component. [LAW]
The model

Every part is a whole

This is where the name comes from. A holon is a thing that is simultaneously a whole and a part — and in Holon that isn’t philosophy, it’s the data model. A Screen is just the top-level component: same abstraction, highest node in the tree. Everything below it behaves identically.

A component’s scope comes from where it was born — workspace-global, project-global, screen-level, or nested inside another component. Scope decides where it can be reused; ownership is always singular and always clear.

The result

An honest tree, zero maintenance

The tree you navigate is never a diagram someone had to keep up to date. It is a direct, live reflection of the design: atoms, molecules, organisms — formed for you as you build, not by you after hours.

You will never componentize anything again. You’ll just build, and the structure will already be there.
