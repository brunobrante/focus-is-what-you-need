Plate 05 history with ownership

Crowns that move.

Keep as many versions of a screen or component as you need — linked to the originals, or copied and free. Promotion is never a label swap.

the problem

“final-final-2” is not a versioning model.

Canvas tools do history by duplication: pages of stale frames pretending to be versions, none of them knowing what depends on what. Duplicate a screen, edit a copy, delete the wrong one — and discover the “copy” secretly shared its parts with the original.

The question a duplicated frame can never answer: who owns this component now?

the clade way

Every version copies the frame. You choose what the insides mean.

Linked — the inner components become live instances of the originals: edit a master, and the original and the version both reflect it. Copy — the version is fully independent: its components are re-created as its own, with no link back. Editing or deleting one affects this version and never the original. That’s law, not a setting.

A dedicated Versions canvas puts any version side by side with the main for honest comparison.

how the component model dissolves it

Promote a version, and components move with the crown.

Any version can be made the main. When it happens, the promoted version becomes the real owner of its components; the old main becomes an ordinary version holding instances that point at the new one.

The link survives — editing the new main still reflects everywhere — but deleting the old version can never empty the new main. A main that secretly depends on its past would be a bug, not a model.
