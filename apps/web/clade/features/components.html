Plate 01 the founding law

Born, not made.

In Clade you never declare a component. Structure is the declaration — the moment an element contains another, a component exists: openable, editable, alone.

the problem

In Figma, a component is a ceremony.

Pick the pattern. Right-click. “Create component.” Name it. File it into a library. Publish. Maintain it forever — and do it all again for the next one. The structure you already built means nothing until you bless it.

So most of a real design never becomes components at all. The ritual costs too much, the library drifts from the file, and the file drifts from the truth.

the clade way

One rule: if an element has a child, it is a component.

There is no “componentize this” step. Nest a text inside a div and the div is a component — automatically, instantly, with no badge to apply. An element with no children is just an element: a lone label, an empty box. The tree stays honest.

Your component tree is a direct reflection of what you actually built — not a curated museum beside it.

how the component model dissolves it

Your structure is your library.

Because components form from structure, the library can never drift: it is the design, seen from above. Every project surfaces all components anywhere inside it — in screens, inside other components, everywhere — with zero filing on your part.

And every one of them can be opened on its own, at true size, the moment it exists.

<div id="button">⊃<div id="text">→button · component

the result

Zero ceremonies. Zero drift.
